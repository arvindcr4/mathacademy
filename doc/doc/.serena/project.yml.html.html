  <a class="pilcrow" href="#section-9" id="section-9"></a>
</div>
<p>list of tool names to exclude.
This extends the existing exclusions (e.g. from the global configuration)</p>
<p>Below is the complete list of tools for convenience.
To make sure you have the latest list of tools, and to view their descriptions,
execute <code>uv run scripts/print_tool_overview.py</code>.</p>
<ul>
<li><code>activate_project</code>: Activates a project by name.</li>
<li><code>check_onboarding_performed</code>: Checks whether project onboarding was already performed.</li>
<li><code>create_text_file</code>: Creates/overwrites a file in the project directory.</li>
<li><code>delete_lines</code>: Deletes a range of lines within a file.</li>
<li><code>delete_memory</code>: Deletes a memory from Serena's project-specific memory store.</li>
<li><code>execute_shell_command</code>: Executes a shell command.</li>
<li><code>find_referencing_code_snippets</code>: Finds code snippets in which the symbol at the given location is referenced.</li>
<li><code>find_referencing_symbols</code>: Finds symbols that reference the symbol at the given location (optionally filtered by type).</li>
<li><code>find_symbol</code>: Performs a global (or local) search for symbols with/containing a given name/substring (optionally filtered by type).</li>
<li><code>get_current_config</code>: Prints the current configuration of the agent, including the active and available projects, tools, contexts, and modes.</li>
<li><code>get_symbols_overview</code>: Gets an overview of the top-level symbols defined in a given file.</li>
<li><code>initial_instructions</code>: Gets the initial instructions for the current project.
Should only be used in settings where the system prompt cannot be set,
e.g. in clients you have no control over, like Claude Desktop.</li>
<li><code>insert_after_symbol</code>: Inserts content after the end of the definition of a given symbol.</li>
<li><code>insert_at_line</code>: Inserts content at a given line in a file.</li>
<li><code>insert_before_symbol</code>: Inserts content before the beginning of the definition of a given symbol.</li>
<li><code>list_dir</code>: Lists files and directories in the given directory (optionally with recursion).</li>
<li><code>list_memories</code>: Lists memories in Serena's project-specific memory store.</li>
<li><code>onboarding</code>: Performs onboarding (identifying the project structure and essential tasks, e.g. for testing or building).</li>
<li><code>prepare_for_new_conversation</code>: Provides instructions for preparing for a new conversation (in order to continue with the necessary context).</li>
<li><code>read_file</code>: Reads a file within the project directory.</li>
<li><code>read_memory</code>: Reads the memory with the given name from Serena's project-specific memory store.</li>
<li><code>remove_project</code>: Removes a project from the Serena configuration.</li>
<li><code>replace_lines</code>: Replaces a range of lines within a file with new content.</li>
<li><code>replace_symbol_body</code>: Replaces the full definition of a symbol.</li>
<li><code>restart_language_server</code>: Restarts the language server, may be necessary when edits not through Serena happen.</li>
<li><code>search_for_pattern</code>: Performs a search for a pattern in the project.</li>
<li><code>summarize_changes</code>: Provides instructions for summarizing the changes made to the codebase.</li>
<li><code>switch_modes</code>: Activates modes by providing a list of their names</li>
<li><code>think_about_collected_information</code>: Thinking tool for pondering the completeness of collected information.</li>
<li><code>think_about_task_adherence</code>: Thinking tool for determining whether the agent is still on track with the current task.</li>
<li><code>think_about_whether_you_are_done</code>: Thinking tool for determining whether the task is truly completed.</li>
<li><code>write_memory</code>: Writes a named memory (for future reference) to Serena's project-specific memory store.</li>
</ul>
